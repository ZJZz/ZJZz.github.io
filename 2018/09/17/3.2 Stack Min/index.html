3.2 Stack Min

思路

题意理解

设计一个栈，除了Push和Pop操作，还有一个返回最小元素的函数，Push,Pop和Min都应该在O(1)时间内完成。

分析1

题目限定在O(1)时间内完成，应该在栈的设计上多考虑，应该满足某种特殊结构。

元素应该在入栈的时候就被安排好了位置，并且随着元素的增多，这些位置相对不变才能在O(1)时间内返回。

这个问题的关键一点是:Pop返回最近入栈的元素，Min返回栈中最小的元素，这两个性质并没有直接的联系。

所以能改动的地方就是Push这个点。

但是都要求在O(1)时间内完成，所以Push的时候不能去遍历整个栈来找到最小元素的位置。

暂时没有头绪，回归到栈本身的定义上来想，只允许一端出入元素，满足FILO。

如果在Push的时候和之前最小的元素比较谁是Min，那么在当前最小元素出栈的时候,如何回到之前的最小元素？

Hint 27

Observer that the minimum element doesn’t change very often. It only changes when a smaller element is added, or when the smallest element is popped.

分析2

一直寄希望于在一个栈上实现，首先栈内元素的顺序不可以改变，必须满足FILO，所以需要Min记录当前的栈内元素的顺序，而且栈内元素随着Push和Pop操作在动态的变化，那么保存顺序的这个额外空间也需要动态调整顺序。

和同学讨论后发现可能需要额外的空间来实现。这个空间可以保存输入元素的大小顺序，或者当前状态的最小元素。

Hint 59

What if we kept track of extra data at each stack node ? What sort of data might make it easier to solve the problem ?

分析3

关于extra data, 我的想法是建立另一个栈，不同的是这个栈的top为当前状态的最小元素。

比如输入顺序为3，1，2, 4。那么保存最小元素的栈为3, 2, 1。在入栈的时候要和栈顶元素比较，比top大的不入栈。 当正常栈的出栈元素为最小元素时，额外栈才pop

Hint 78

Consider having each node know the minimum of its “substack” (all the elements beneath it, including itself).

分析4

“substack”是什么意思？( all the elements beneath it, including itself )

实现

大纲

  1. 用两个栈来实现，第一个栈来存储输入，另一个栈用来存储第一个栈中状态下的最小值。
  2. 第一个栈正常进行操作，另一个栈保存当前输入状态下的最小值。当Push的值比最小值小时，把这个值也Push进最小栈。当Pop的值比最小值小时, 最小栈也Pop。其它状态下最小栈状态不变。
