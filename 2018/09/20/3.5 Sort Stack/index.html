3.5 Sort Stack

思路

题意理解

按升序(top为最小元素)的顺序重排一个栈，可以借用一个额外的栈，但不允许拷贝复制栈中的元素到另一个数据结构。

分析1

问题可以简化为两个只允许一端出入并且FILO,如何相互操作使原来栈中的元素变得有序？

整体有序应该由局部有序慢慢推进。可是两个限定了FILO序的结构，并且只允许在一端操作，如何做到序需要改变的时候调整？

总感觉两个栈在相互还原顺序，没有找到方法如何破坏这些限制去实现这个题的顺序。

Hint 15

One way of sorting an array is to iterate through the array and insert each element into a new array in sorted order. Can you do this with stack ?

分析2

提示的意思也许是在说如何在栈里做到在插入另一个辅助数据结构的时候按序插入？

如果辅助结构的包含全部元素的话，那么在栈中就应该是最大元素在上，最小在下。然后按顺序插入原来的栈中。

但是我遇到的问题是，我按有序的插入辅助栈，发现序无法再保持下去的时候就需要放回原来的栈中，这样之前构造的序就还原了。

Hint 32

Imagine your secondary stack is sorted. Can you insert elements into it in sorted order ? You might need some extra storage. What could you use for extra storage ?

Hint 43

Keep the secondary stack in sorted order, with the biggest elements on the top. Use the primary stack for additional storage.

参考代码思路理解

借助第三方变量存储Pop出来的元素，可以改变两栈相互还原的循环。

另一个神奇的方法是归并排序的思想来做。

实现

大纲

  • 方法1：双栈实现
  1. 主要是借助另一个栈和辅助空间来调整原来栈的顺序，将所有元素在辅助栈中按从大到小排序
  2. 重点是如何调整顺序的判断条件
  3. 当辅助栈为空时，应该Push进元素
  4. 当辅助栈不为空时，Push的时候就需要分情况处理
    • 当原始栈Pop出来的元素大于等于辅助栈的栈顶元素，就Push进辅助栈
    • 当原始栈Pop出来的元素小于辅助栈的栈顶元素时，就需要Pop辅助栈的元素至原始栈中
  5. 当原始栈为空时，辅助栈的元素应按从大到小的顺序排列。
  • 方法2: 归并排序
