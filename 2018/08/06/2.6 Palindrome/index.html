2.6 Palindrome

思路

题意理解

写一个算法判断链表是否回文

感想

觉得单链表的本质是只能单向且顺序访问的序列。

分析1

回文的判断就是两边读起来一样，在数组中的做法是从两边同时访问。但是单链表的性质限制了从两端开始同时访问的这一思路。

有没有其他的做法达到同样的效果？

分析2

如果链表结构本身无法达到操作目的的话，就只能借助外部力量了。我现在的想法是借助一个数组，遍历链表。当链表遍历前半部分的是否存在数组里，遍历后半部分的时候就访问数组对应的位置上是否有对应的值。

Hint 5

A palindrome is something which is the same when written forwards and backwards. What if you reversed the linked list ?

分析3

我也想过翻转链表，是翻转前的顺序保存一次，翻转后的顺序保存一次，然后比较两个是否相等吗？

Hint 13

Try using a stack.

分析4

果然涉及到逆序的时候就和递归，栈有关。

我想到栈的用法时链表中的元素一半入栈，另一半不入，当分成两半后，链表继续访问，存入栈的部分开始出栈。

Hint 29

Assume you have the length of the linked list. Can you implement this recursively ?

分析5

递归到一半然后开始return ?

base case 是 index = n /2 ?

这样不行，中间相邻的两个元素还可以判断。当递归返回的前面的元素的时候就会断掉。那么递归函数该怎么设计？

Hint 61

In the recursive approach ( we have the length fo the list ), the middle is the base case: isPalindrome(middle) is true. The node x to the immediate left of the middle: What can that node do to check if x->middle->y forms a palindrome ? Now suppose that checks out. What about the previous node a ? If x->middle->y is a palindrome, how can it check that a->x->middle->y->b is a palindrome ?

分析6

Hint 61在说我在分析5中想不明白的地方。

我的一个想法是在中间元素作为base case。返回的时候返回中间右边的那个结点middle.next。这样右边这半就有了线索，没有断掉。然后开始比较，相等就继续后移，每次都返回多加一层next。这样就会把右半部分都遍历完。

以null作为具有回文序列的标志。因为返回null说明右半部分遍历完了。

Hint 101

Go back to the previous hint. Remember: There are ways to return multiple values. You can do this with a new class.

分析7

突然明白了为什么上一题，2.5题逆序情况要用一个结构体辅助。就是为了要返回多个值。

但是我在分析6中的方法不可行吗？
