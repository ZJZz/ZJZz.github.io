2.8 Loop Detection

思路

题意理解

给出一个链表，里边有循环结点，就是它的next指向前一个结点，产生一个循环。实现一个算法，返回这个循环开始的结点。

例如：

Input: A->B->C->D->E->C [和前面一样的C]
Output: C

分析1

首先这里应该是引用相同，而不是值相同。

导致的结果是这里的tail不为null

此时是无法计算长度的

突然想到了引用计数，用hash table，为每个结点建立一个引用计数，第一个为个数为2的结点就是循环开始的结点。

是否有利用循环本身的方法？

Hint 50

There are two parts to this problem. First, detect if the linked list has a loop. Second, figure out where the loop starts.

分析2

用hash table方法的话可以把这两部分同时解决。

Hint 69

To identify if there’s a cycle, try the “runner” approach. Have one pointer move faster than the other.

分析3

用runner方法的话是可以用来检测是否有循环的，跑的快的runner是会再和走的慢的指针再相遇的。

剩下是runner应该快几个结点？和如何找出循环开始的地方？

倒是有个效率不高的办法，固定一个，让runner遍历完链表，如果没有相遇，就移动慢的，再让runner从慢的结点开始继续遍历。

不太可能，应为进入循环后无法知道停止的地方。

慢的那个指针也必须进入循环，两个指针才有可能相遇。

问题是相遇的位置也可能不是循环开始的地方。

这个问题类似一个带回旋的过山车，进去以后会一直在回旋里转，不能两个都进去，必须留一个在外面作为标记起点开始的地方。

Hint 83

You can use two pointers, one moving twice as faster as the other. If there is a cycle, the two pointers will collide. They will land at the same location at the same time. Where do they land ? Why there ?

分析4

快两倍速度会产生什么关系？？ 我不是很懂。

能保证在循环开始的地方遇到吗？因为两倍指针可以访问循环中的任何位置。

Hint 90

If you haven’t identified the pattern of where the two pointers start, try this: Use the linked list 1->2->3->4->5->6->7->8->9->?, where the ? links to another node. Try making the ? the first node. Then make ? the node 2. Then the node 3. Then the node 4. What is the pattern ? Can you explain why this happens ?

分析5

我还是不知道当两指针相遇的时候，一个是另一个的两倍速度，如何保证相遇的时候就是循环开始的地方？

参考代码思路理解

这道题的思路很巧妙，看了参考代码后还理解了很长时间，其中还把fast的位置理解错误，导致规律没有发现出来。

大题思想是，设置两个指针，其中fast快于slow指针2倍速度，但都是从起点head开始。

遍历，当fast=slow时，停止，把slow重新调回head的位置，此时slow距循环开始的距离，和fast相遇的位置开始，fast也一步速度遍历到循环开始的结点的距离相同。

此时让fast和slow以同样的速度开始遍历,当它们再次相遇的时候就是循环开始的位置。

具体原理的证明我不是很明白，经学弟们的努力，才知道这叫Floyd判圈法
