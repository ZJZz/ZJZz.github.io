1.8 Zero Matrix

思路

题意理解

题目的意思是如果矩阵中有一个元素为0，那么将这个元素所在的行或列都设为0，如果我没理解错的话。

分析1

乍一看好像不怎么难，只要找到某个元素为0，直接遍历这个元素所在的行和列，都置为0就可以了。会有什么陷阱吗？

Hint 17

If you just cleared the rows and columns as you find 0s, you’d likely wind up clearing the whole matrix. Try find the cells with zeros first before making any changes to the matrix.

我不明白为什么清除我找到的0，就会把整个矩阵清除为0，除非这些0所在行和列覆盖了整个矩阵，这样就会使得整个矩阵都为0。

Hint 74

Can you use O( n ) additional space instead of O( n^2 ) ? What information do you really need from the list of cells that are zero ?

我不是很明白为什么要用O(n)的额外空间，我觉得根本不用其他空间，直接把数设为0就可以了，为什么还要额外空间呢？

Hint 102

You probably need some data storage to maintain a list of the rows and columns that need to be zeroed. Can you reduce the additional space usage to O(1) by using the matrix itself for data storage ？

分析2

我突然明白了为什么清除找到的0，就会把整个矩阵清除为0，因为如果找到0就马上清除的话，这个0所在的行或列都变为0,所以下次再遍历的时候就会遇到因为清除之前的0变成的0。这会产生联代效应，最终把整个矩阵都变为0。

(由于在脑中模拟了清除0的几种情况，突然明白之前没有想明白的地方。这对以后思考问题也很有帮助)

这样的话就需要先保存哪些元素为0，而且需要知道哪些0同行或同列。

分析3

可以用hash table来存储待修改的位置，key是行，value是列。这样可以实现O(n)的附加空间。

如果再按Hint 102的思路往下考虑，按行从上到下，当找到一个0时，用一个变量来存储第一个0的行和列，仍然用hash table。第二个0找到后，将这一行置零，在这一行的末尾2个元素存储上一个0的行，在这个位置存储上一个0的列。这样逐行寻找，就不会把已经设为0和没被找到过的0弄混。

分析4

在这种情况下有可能出现同一行或列对应多个0，hash table的键可能会有重复，但是这个问题的特点是有0就全为0，即使重复也不影响结果。

参考代码2思路理解

看了参考代码才发现，分析问题要分析出问题的本质，而不是在一些表面的东西做一些琐碎的工作。在分析4中我得出了只要一行或一列中有0，就不必再看这一行或列的其它元素，但是参考代码更进一步利用了这个性质，也更能体现这道题的本质。

参考代码的思路是利用第一行和第一列作为整个矩阵0元素的标尺，首先判断第一行和第一列本身是否有0。然后再判断矩阵其它元素是否为0，有0就将对应的第一行和第一列置为0。最后利用之前得到的第一行和第一列是否为0的信息，在对第一行和第一列0化。

因为一个元素为0，这个元素的对应的第一行和第一列肯定为0，所以第一行的元素不必全部保留，可以作为标尺。但最后要再检查第一行或第一列本身为0的情况。
