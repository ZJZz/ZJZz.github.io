1.4 Palindrome Permutation

思路

题意理解

给定一个字符串，生成这个字符串的所有排列中具有回文性质的排列

分析1

有一点觉得很难，生成所有排列，这好像是一个复杂度很高的问题。

回文的判断倒是觉得有办法去做，字符串前后两端设置一个标志位，分别沿前和后去遍历字符串。不相等直接返回False。

漏了一点，中间有空格的话不影响字母之间的回文性质。

Hint 106

You do not have to - and should not to - generate all permutations. This would be very inefficient.

分析2

既然Hint说没必要产生所有排列，看来这个问题具有一些特殊性质，再好好想一下。

发现一个规律，如果一个字符串具有回文性质的排列，那么字符串中字母的个数有一定的规律：

  • 如果总字母个数为偶数的话，每个字母必然成对出现。
  • 如果总字母个数为奇数的话，有1个个数为1的字母，它会在字符串中间的位置。剩下的字母全为偶数，且成对出现。

只有在个数上具有上面2个规律的字符串才能构成回文序列。

下面考虑生成回文序列的个数：

  • 字符个数为偶2n，个数应该为字符个数一半的排列数即，n!
  • 字符个数为奇2n+1,个数也为n!

又重现看了一下题目,好像并没有要求给出个数，只是判断是否存在。如果是这样的话，问题基本上可以解决了。再看一下没用的Hint说了什么。

Hint 121

What characteristics would a string that is a permutation of a palindrome have ？

分析3

Hint 121应该就是在说我在分析2中发现的两个规律

Hint 134

Have you tried a hash table ？You should be able to get this down to O(n) time.

分析4

Hint 134 应该涉及到具体的实现细节。如果用hash table的话，按之前的想法，记录每个字符的个数，统计是否都为2。但是要在O(n)时间里完成的话，就应该“初见增1，再见减1”的方法。

等一下！

“初见增1，再见减1”的方法适用于字符的个数为2的情况，如果为4或者6呢？

可以这样，当这个值为0时，再遇到就增1，当为1时就，遇见就再减1。

最后的hash table应该有两种最终形态，当字符串为偶数时，全为0；当字符串为奇数时，只有1个为1，其它全为0。

但是这样做在遍历完字符串后，又要遍历一遍hash table。有没有遍历完字符串就能出结果的方法?

Hint 136

Can you reduce the space usage by using a bit vector ?

分析5

在分析4中的最终结果时,我想到了用位运算，个数为偶数的时候还可以方便一些，判断结果是否为0。对于奇数该怎么操作? 最终剩下的1位置不固定,位运算操作的结果也不唯一。

想到的一个办法是记录这个1的位置，最后结果与1进行与操作前，先把这个1移到最低位。

但是具体操作还不是很清楚。

代码思路1

遍历字符串。

① hash table

使用dict来存储遇到的字符，忽略26字母之外的其它字符，dict中的value只有两种值0和1，0增1减。最后分奇偶两种情况来判断hash table的最终状态。

② 位运算

思路和hash table的一样，只是对应的操作不知道如何在位运算中体现。

比如刚开始初始化是用0还是1？

遇到字符时该怎么操作? 左移1吗? 第一次还好说，第二次移位的时候，会把前面的1也移动，这样位置不就错乱了吗？

突然有个想法，在每个位置遇到字符时，就设该位为1,其它位为0的位串进行与运算。再遇到时。。。

分析6

中午去吃饭的路上想了关于位运算的操作，这里总结一下：

以0初始化,每次遇到一个字符算它与’a’的距离，以这个距离作为1左移的位数。然后拿这个左移后的数与0进行异或。如果该位为0，与1异或后变为1。如果该位为1与1异或后变为0。

这个想法不是很完善，有两个问题：

  1. 字符长度为偶数最后结果为0还好说，奇数位的数最后结果不唯一，该怎么确定？
  2. 在进行异或的时候，其它位的值也会变化。如果其它本来为0，与0异或后，还是0;如果其它位为1，与0异或后，还是1。不，其它位不受影响。

现在就剩奇数位这个问题了。。。

晚饭过后想了一下还是没有想出来，如果是奇数能构成回文序列的字符串，它们的位串在进行运算后，应该只剩下1个1，但是这个1可能出现在任意一个位置上，我没想出来用什么位串来和这个结果进行位运算。也考虑过不管位置，只找1个位串和结果运算后可以得出1的个数，最后还是没有想出来拿什么位串来操作。

问了学姐，我懂了这段代码表达的意思。因为对二进制的减法不熟悉，所以刚开始无法理解bitVector - 1的含义。

bitVector - 1在二进制表示中，最低位如果为0,那么需要向高位借位，但二进制中只有0和1,在最近的高位1借位后，之前的所有位都与原来的数相反，但高位如果还有1的话就保持不动。这样再与bitVector进行与运算，如果原来只有1个1的话结果就为0，如果不止1个1的话就不为0。也就是这段代码表达的意思。
