1.2 Check Permutation

思路

  • Hint #1: Describe what it means for two strings to be permutations of each other. Now, look at that definition you provided. Can you check the strings against that definition ?
  • Hint #84: There is one solution that is O(N log N) time. Another solution uses some space, but is O(N) time.
  • Hint #122: Could a hash table be useful ?
  • Hint #131: Two strings that are permutations should have the same characters, but in different orders. Can you make the orders the same ?

分析1

两个字符串互为排列要满足3个条件：

  1. 长度相等
  2. 组成字符串的元素相同
  3. 顺序可以相同，也可以不同

这里有一点容易忽视，就是每个字符可以不止出现1次，可以出现多次。

发现这里两个字符串互为排列与顺序无关，这里只要每个字符出现的次数相等就可以

想法1

也许这里可以使用在Is Unique里的想法：总共2个字符串，为每个字符和它出现的次数创建一个词典，循环遍历完后，看相同的字符个数是否相同，不相同就不互为排列。这样应该是O(n)的复杂度。

疑惑1

还是不知道O(N log N) 是用的什么方法

分析2

由分析1里最后一点，既然最终的判断与字符串的顺序无关，只要每个字符个数相等，那么可以对字符串进行处理，比如排序，让相同的字符“挤”在一起,这样就省去了创建词典的空间和时间。

想法2

先对两个字符串进行排序，排序后相同的字符会在一起，用两个变量分别表示两个字符串中相同字符的个数，如果相等继续遍历，如果不等就返回False。

参考代码1思路理解

不必用想法2中的两个变量存储每个字符的个数，可直接判断两个字符串是否相等。

参考代码2思路理解

不对字符串进行排序，利用两个字符串相等这一特性，类似键词典的思想，首先遍历第一个字符串,统计每个字符的个数，再遍历第二个字符串，出现对应字符就减去1，最后判断是否为负值，有负说明字符个数不相等，直接返回False

  • 最后遍历键值对是否全为0，又不为0的就返回false
