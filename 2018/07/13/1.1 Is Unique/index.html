1.1 Is Unique

思路

  • Hint #44: Try a Hash Table
  • Hint #117: Could a bit vector be useful ?
  • Hint #132: Can you solve it in O(N log N) time ? What might a solution like that look like ?

想法1

创建一个dict，字母表中的每一个字母作为key,然后遍历输入的字符串，每个字母出现一次就把对应的value加1。字符串遍历完后，看dict是否里面有大于等于2的value,如果有就返回False,没有就返回True。

疑惑1

  • dict是否算Hash Table ?
  • 没有用到Hint中给到的bit vector,如果要用的话该怎么用？
  • 提示中O(n log n)的复杂度是怎么样的？或者说能出现nlogn这种复杂度的都做了什么操作？

参考代码1思路理解

看了给出的solution的解法，发现想法1实在是笨拙。solution的方法避免了为每一个字母创建键值对，利用了bool数组，初始化全为False,当遇见一个字符时变为True,下次再遇到时，说明这个字符串不Unique，可直接返回False。当遍历完整个字符串的时候，可以返回True。

参考代码1思路总结

  • 这个题目没有要求保存每个字母出现的个数，只看每个字母是否唯一，所以True和False可以满足需求，刚开始设为False,出现1次设为True，再利用这个位置的值，再出现就直接返回False。
  • 也不需要遍历完整个字符串，如果前面有重复的，可直接返回Fasle
  • 代码假设长度至多为128是因为ASCII总共128个字符

参考代码2思路理解

代码2使用了位向量，整体思路应该和代码1差不多，但是位运算我不熟，所以先复习一下位运算。

位运算快速复习

刚开始checker=0，val计算出的是ASCII码中每个字母与a的距离,第一次循环时，checker=0不会执行if中的语句。而是执行循环体中的最后一句。

以16位为例：把 0000 0000 0000 0001 中的低位的1左移val位后，再与checker进行或运算。这样的结果使得checker中有1的那位表示这个位置的字符出现过。再进行循环后，if中的条件语句表示如果与a同样距离的字符再出现过就返回False。

参考代码2思路总结

位向量更省空间和简洁，需要体会0和1这两个特殊的值在位运算中的作用：

  • 0的位全为0，可以和其它数字进行or运算
  • 1的表示中只有1个1，而且在低位，可以进行左移运算。

方式2 位运算

  1. 大纲

①首先需要一个0
②然后看到1个字符需要算出1个距离，然后1左移这个距离，再与刚开始的0进行或运算
③按说如果每个字符出现1次，对应的位上应该是0。拿单个1去和这个数进行与运算，如果只出现了1次，与运算后的结果应为0。如果不只1次出现，那么结果应该就不为0。以此为判断条件，返回函数的判断结果
