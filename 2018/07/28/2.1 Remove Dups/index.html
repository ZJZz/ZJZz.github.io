2.1 Remove Dups

思路

分析1

第一个要想明白的问题，怎么在链表中找到重复的节点，并且要把所有重复的都找到？

假设为单链表。一个简单的思路是，设两个指针，1个来指定当前的元素，另1个用来遍历链表，当发现相同的就删除节点。这样下来时间复杂度是O( n^2 )。有没有更巧妙的方法？

明确一下问题:
假设a为当前的节点，a之前的节点都和a不重复。那么从a往后的元素中在任意位置上可能出现任意个数与a相同的节点。

对链表的操作要考虑两头和中间

Hint 9

Have you tried a hash table ? You should be able to do this in a single pass of the linked list.

分析2

看了Hint 9, 大概明白想法了。借助hash table, 而不是两个指针来实现遍历和发现重复结点。

Hint 40

Without extra space, you’ll need O( N^2 ) time. Try using two pointers, where the second one searches ahead of the first one.

分析3

现在有了完整的想法：

  • 用hash table

    遍历整个链表，并对每个内容进行统计，当这个值在hash table中个数为1时就删除。时间复杂度为O( n )

  • 不用hash table

    利用两个指针，一前一后，后指针移动1个，前面遍历整个链表，遇到与后指针相同的值时就删除。当后指针遍历完整个链表后就结束。
